# Requirements: Flow×Stock PKM App — Phase 1 (Prototype)

## Goal

Slackライクなチャンネルベースのフロー管理を個人用に実現する。
Phase 1 は「チャンネル＋メッセージ投稿＋全文検索」に絞り、ローカルで動くプロトタイプとして構築する。

## Constraints

- プロトタイプ — ローカルで動けば十分
- ユーザーは1人想定、認証不要
- SQLite にデータ保存（ローカルファイル）
- Next.js の Server Actions を使用、別バックエンドは作らない
- 画像対応は Phase 1 スコープ外

## Functional Requirements

### チャンネル管理
- [ ] FR-1: チャンネルの作成（名前・説明を入力）
- [ ] FR-2: チャンネル一覧をサイドバーに表示
- [ ] FR-3: チャンネルの削除
- [ ] FR-4: チャンネルの編集（名前・説明の変更）

### メッセージ投稿
- [ ] FR-5: チャンネル内にテキストメッセージを投稿
- [ ] FR-6: メッセージのMarkdownレンダリング（インライン表示）
- [ ] FR-7: メッセージの時系列表示（新しいものが下）
- [ ] FR-8: メッセージの編集・削除

### スレッド
- [ ] FR-9: メッセージへのスレッド返信
- [ ] FR-10: スレッドの展開表示（サイドパネル）

### リアクション
- [ ] FR-11: メッセージへの絵文字リアクション追加・削除
- [ ] FR-12: `:記法` によるインライン絵文字入力（サジェスト付き）

### 検索
- [ ] FR-13: グローバル検索モーダル（Cmd+K で起動）
- [ ] FR-14: メッセージの全文検索（SQLite FTS5）
- [ ] FR-15: 検索結果から元メッセージへのジャンプ

### カレンダービュー
- [ ] FR-16: チャンネルごとの1ヶ月カレンダービュー（投稿があった日にドット表示）

## Non-Functional Requirements

- [ ] NFR-1: レスポンシブWebデザイン（モバイルブラウザ対応）
- [ ] NFR-2: ローカル環境で完結（外部サービス依存なし）
- [ ] NFR-3: メッセージ投稿は体感的に即時（< 300ms）

## Tech Stack

| レイヤー | 採用 | 理由 |
|---------|------|------|
| Framework | Next.js 15 (App Router) | Server Actions でバックエンド不要 |
| ORM | Drizzle ORM | 軽量、型安全、SQLite対応 |
| DB | SQLite (better-sqlite3) | ローカルファイル、FTS5内蔵 |
| 全文検索 | SQLite FTS5 | 追加サービス不要 |
| UI | Tailwind CSS + shadcn/ui | 高速開発 |
| Markdown | react-markdown + remark-gfm | インラインレンダリング |

## Out of Scope

- 画像アップロード
- 認証
- クラウドデプロイ
- 昇格・インボックス・ノート管理（Phase 2）
- バックリンク・AIアシスト（Phase 3）
